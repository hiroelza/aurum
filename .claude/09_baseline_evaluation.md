# ベースライン評価の記録ルール

**作成日**: 2025年11月9日
**最終更新**: 2025年11月9日

---

## 📌 このドキュメントの目的

**ベースライン評価**とは、**実投資は行わないが、将来の比較のために現時点での評価を記録すること**です。このドキュメントでは、ベースライン評価の適切な使い方と記録方法を明確化します。

---

## 🎯 ベースライン評価の目的

### 1. 将来の自分への学習記録

**シナリオ**: 現在は投資判断を凍結中だが、市場や投資対象を学習したい。将来、精神的に回復したときに振り返るための記録を残す。

**効果**:
- 「あのとき考えていたことは正しかったか？」を検証できる
- 感情的判断（FOMO、不安駆動）を客観的に評価できる
- 学習の進歩を実感できる

---

### 2. 市場タイミングの比較研究

**シナリオ**: 現在の市場環境（2025年11月）と将来の市場環境（例: 2026年、NISA枠が埋まったとき）を比較したい。

**効果**:
- 「あのとき買っていたらどうなっていたか？」を検証できる
- 市場タイミングの影響を定量的に理解できる
- 「待つこと」の価値を実感できる

---

### 3. 投資哲学の一貫性検証

**シナリオ**: 現在の評価と将来の評価を比較し、自分の投資哲学が一貫しているかを確認したい。

**効果**:
- 感情的な判断の変動を可視化できる
- 投資哲学の進化を追跡できる
- 「なぜ以前はそう考えたのか？」を振り返れる

---

## ⚠️ ベースライン評価と実投資の違い

### ベースライン評価は「記録」であり「判断」ではない

**重要**: ベースライン評価で高スコアが出ても、**それは実投資の根拠にはならない**。

| 項目 | ベースライン評価 | 実投資判断 |
|------|-----------------|-----------|
| **目的** | 学習・記録 | 実際に資金を投入 |
| **実行条件** | いつでも可能 | 投資判断凍結ルール遵守 |
| **リスク** | なし | 資金損失の可能性 |
| **記録先** | `INVESTMENT_PROFILE.md`または`investment_history.md` | `INVESTMENT_PROFILE.md`の「現在の資産配分」 |
| **評価後の行動** | 記録して終了 | 実際に購入 |

## 🔴 絶対に避けるべき誤用パターン

### 1. ベースライン評価の高スコアを実投資の根拠にする

**シナリオ**: ベースライン評価で70点が出た。「これは買いだ！」と考え、実際に投資する。

**問題点**:
- **投資判断凍結ルール**を無視している（CES-D 33, GAD-7 14-15）
- ベースライン評価は「学習目的」であり、実投資の根拠ではない
- 高スコアは「現時点での評価」であり、将来の成功を保証しない

**正しい対応**:
- ベースライン評価の結果を記録し、**実投資はしない**
- CES-D < 10, GAD-7 < 5 に回復するまで待つ
- 将来、精神的に回復したときに再評価する

---

### 2. ベースライン評価を繰り返し実行してスコアを上げようとする

**シナリオ**: 最初の評価で40点だった。「もっと高いスコアが欲しい」と考え、ウェイトを調整して再評価する。

**問題点**:
- **スコア操作**が目的になっている
- ベースライン評価の本質（現時点での客観的評価）を無視している
- FOMO（乗り遅れる恐怖）や不安駆動の行動

**正しい対応**:
- 最初の評価結果をそのまま記録する
- スコアが低い理由を分析し、学習する
- 「待つこと」の価値を認識する

---

### 3. ベースライン評価を実投資の「言い訳」にする

**シナリオ**: 「ベースライン評価で高スコアだったから、実投資しても大丈夫だろう」と考える。

**問題点**:
- ベースライン評価は実投資の承認ではない
- 投資判断凍結ルールを迂回しようとしている
- 感情的判断を正当化するための「言い訳」

**正しい対応**:
- ベースライン評価は「記録」であり、実投資の根拠ではないと認識する
- 投資判断凍結ルールを遵守する
- 実投資したい衝動の背後にある感情（FOMO、不安）を分析する

## ✅ ベースライン評価の適切な使用例

### 例1: 個別株投資の学習目的での評価

**背景**:
- 投資判断は凍結中（CES-D 33, GAD-7 14-15）
- 個別株投資を学びたいが、実投資はしない
- 将来、精神的に回復したときに振り返るための記録を残したい

**実行内容**:
1. 個別株20銘柄を選定
2. 3エージェント評価を実施（デフォルト: 33/33/33、調整後: 15/45/40）
3. 結果を記録（35点 → 43点、投資は推奨しない）
4. 学習内容を文書化（ウェイト調整の効果、FOMOの危険性など）

**記録先**:
- `outputs/reports/working/202511/20251109_stock_analysis/04_final_report.md`
- `INVESTMENT_PROFILE.md` の「## 📊 過去の評価履歴」

**結果**:
- 実投資はせず、記録のみ残す
- 将来（2026年、NISA枠が埋まったとき）に振り返る

---

### 例2: 暴落時の市場タイミング研究

**背景**:
- 市場が暴落した（例: 日経平均 -15%）
- 「今が買い時か？」を学びたい
- 実投資はせず、学習目的での評価を記録

**実行内容**:
1. 暴落時の市場環境を分析（PER, PBR, 配当利回り）
2. 3エージェント評価を実施（インデックスファンドの追加購入を評価）
3. 結果を記録（例: 75点、統計的には買い時だが、感情的判断のリスクあり）
4. 学習内容を文書化（暴落時の心理、統計的分析の限界など）

**記録先**:
- `outputs/reports/working/yyyymm/yyyymmdd_market_crash_study/`
- `INVESTMENT_PROFILE.md` の「## 📊 過去の評価履歴」

**結果**:
- 実投資はせず、記録のみ残す
- 次回の暴落時に参照する

---

### 例3: 投資哲学の一貫性検証

**背景**:
- 2025年11月に個別株を評価（35点、投資推奨せず）
- 2026年、精神的に回復し、NISA枠が埋まったとき、再度評価したい

**実行内容**:
1. 2025年11月の評価を記録（ベースライン評価）
2. 2026年、同じ銘柄を再評価
3. 2025年と2026年の評価を比較
   - スコアの変化
   - 各エージェントの意見の変化
   - 市場環境の変化
4. 学習内容を文書化（投資哲学の一貫性、市場タイミングの影響など）

**記録先**:
- 2025年11月: `outputs/reports/working/202511/20251109_stock_analysis/`
- 2026年: `outputs/reports/working/202601/yyyymmdd_stock_reevaluation/`
- 比較結果: `outputs/reports/completed/yyyymm/yyyymmdd_consistency_check/`

**結果**:
- 投資哲学の一貫性を確認
- 感情的判断の変動を可視化
- 「待つこと」の価値を実感

## 📋 ベースライン評価の実行手順

### Step 1: 評価の目的を明確化

以下のチェックリストで確認：

- [ ] 評価の目的は何か？（学習、市場タイミング研究、投資哲学の検証など）
- [ ] 実投資に使うのか、学習目的か？
- [ ] FOMO（乗り遅れる恐怖）や不安駆動ではないか？
- [ ] 投資判断凍結ルールに違反していないか？（CES-D < 10, GAD-7 < 5）

**目的が「実投資」の場合 → ベースライン評価ではなく、実投資判断として扱う**

---

### Step 2: 評価対象を選定

**対象の種類**:
- 個別株（日本株、米国株）
- インデックスファンド（NISA成長投資枠、特定口座）
- 資産配分の変更（株式比率の調整など）

**注意点**:
- 評価対象を明確に記録する（銘柄名、ティッカー、購入金額など）
- 市場環境を記録する（日経平均、S&P500、為替レートなど）

---

### Step 3: 3エージェント評価を実行

**実行コマンド**:
→ `.claude/06b_three_agent_detailed.md`の「3.2 順次実行（重要判断、時間がある場合）」を参照

**評価内容**:
- デフォルト評価（33/33/33）
- 必要に応じて調整後評価（例: 15/45/40）
- 両方の結果を記録

**評価項目**:
- 各エージェントのスコア（0-100点）
- 各エージェントの意見（賛成理由、反対理由）
- 総合評価（加重平均）

---

### Step 4: 結果を解釈し、学習内容を文書化

**解釈のポイント**:
1. **スコアの意味を理解**
   - 例: 35点 → 投資は推奨しない
   - 例: 75点 → 統計的には優れているが、投資哲学との整合性に注意

2. **各エージェントの意見を読む**
   - hayatoが5点をつけた理由は何か？
   - researcherが45点をつけた理由は何か？
   - japaneseが55点をつけた理由は何か？

3. **学習内容を文書化**
   - 「なぜこのスコアになったのか？」
   - 「自分の判断とエージェントの意見はどう違うか？」
   - 「どのような学びがあったか？」

**記録項目**:
```markdown
## ベースライン評価: [評価対象名]

**評価日**: 2025年11月9日
**目的**: [学習目的、市場タイミング研究など]
**評価対象**: [銘柄名、ティッカー、購入金額など]
**市場環境**:
- 日経平均: 39,500円
- S&P500: 5,900
- USD/JPY: 152円

### 3エージェント評価結果

**デフォルト評価（33/33/33）**:
- hayato: 5点
- researcher: 45点
- japanese: 55点
- **総合: 35点**（投資は推奨しない）

**調整後評価（15/45/40）**:
- hayato: 5点
- researcher: 45点
- japanese: 55点
- **総合: 43点**（投資は推奨しない）

### 各エージェントの意見

**hayatoエージェント（5点）**:
- [反対理由]
- [リスクの指摘]

**researcherエージェント（45点）**:
- [統計的分析の結果]
- [注意点]

**japaneseエージェント（55点）**:
- [税制・ライフプラン上の評価]
- [注意点]

### 学習内容

1. [学んだこと]
2. [気づいたこと]
3. [将来に向けての行動]

### 実投資の判断

**実投資: しない**
- 理由: 投資判断凍結中（CES-D 33, GAD-7 14-15）
- 将来の行動: 精神的に回復したとき（CES-D < 10, GAD-7 < 5）に再評価
```

---

### Step 5: 記録先を決定

**記録先の選択**:

| 記録の種類 | 記録先 | 理由 |
|-----------|--------|------|
| **簡易記録**（結果のみ） | `INVESTMENT_PROFILE.md` の「## 📊 過去の評価履歴」 | すぐに参照できる |
| **詳細記録**（分析・学習内容含む） | `sources/profile/investment_history.md` | 詳細な履歴管理 |
| **作業中ファイル** | `outputs/reports/working/yyyymm/yyyymmdd_テーマ/` | 調査プロセス全体を保存 |
| **最終報告書** | `outputs/reports/completed/yyyymm/yyyymmdd_テーマ/` | 完成した分析レポート |

**推奨**:
- `INVESTMENT_PROFILE.md`に簡易記録（結果・判断のみ）
- `outputs/reports/working/`に詳細分析（プロセス・学習内容）
- 重要な学びは`sources/profile/investment_history.md`にも記録

## 🔍 ベースライン評価の妥当性チェックリスト

ベースライン評価を実行する前に、以下をチェック：

### ✅ 適切な使用

- [ ] 評価の目的が明確（学習、市場タイミング研究、投資哲学の検証など）
- [ ] 実投資はせず、記録のみ残す
- [ ] 投資判断凍結ルール（CES-D < 10, GAD-7 < 5）を遵守
- [ ] 評価結果を「将来の比較」のために記録する
- [ ] 学習内容を文書化する
- [ ] FOMO（乗り遅れる恐怖）や不安駆動ではない

### ❌ 不適切な使用

- [ ] ベースライン評価の高スコアを実投資の根拠にする
- [ ] スコアを上げるために繰り返し評価する
- [ ] 投資判断凍結ルールを迂回するために使う
- [ ] 実投資の「言い訳」にする
- [ ] FOMO（乗り遅れる恐怖）や不安駆動が動機

**1つでも❌にチェックが入る場合 → ベースライン評価は見送る**
