## サブエージェント間の議論フレームワーク ⭐

**適用シーン:**
- 詳細を深掘りする必要がある内容
- 複数の視点からの評価が必要な場合
- 投資判断など重要な意思決定

---

## 議論の4フェーズ

### 1. 共有（Share）
- 各エージェントが持っている情報を共有
- 前提条件・制約条件の確認
- 共通認識の形成

### 2. 発散（Diverge）
- 各エージェントが異なる視点から分析
- 多様なアイデア・選択肢を洗い出す
- 批判せずにアイデアを出し切る

### 3. 収束（Converge）
- 出されたアイデアを評価・絞り込み
- 共通点・相違点を整理
- 優先順位付け

### 4. 決定（Decide）
- 最終的な結論を導出
- 根拠を明確にする
- 次のアクションを決定

---

## ファイル出力ルール

**議論の各フェーズを記録:**
```
outputs/reports/working/202511/yyyymmdd_テーマ/
├── 議論_[テーマ]_phase1_共有.md
├── 議論_[テーマ]_phase2_発散.md
├── 議論_[テーマ]_phase3_収束.md
└── 議論_[テーマ]_phase4_決定.md
```

**最終結論の配置:**
```
outputs/reports/completed/yyyymmdd_テーマ/
└── [テーマ]_最終結論.md
```

## 使用例

### 投資判断の議論

**テーマ**: S&P500追加投資の判断

**Phase 1: 共有**
- 現在の資産状況（総額、ポートフォリオ比率）
- 投資原則（INVESTMENT_PROFILE.md）
- 市場状況

**Phase 2: 発散**
- **Agent 1（哲学）**: 原則に沿っているか
- **Agent 2（統計）**: リスク・リターンの評価
- **Agent 3（税制）**: 税務上の影響

**Phase 3: 収束**
- 各エージェントの評価を統合
- 共通する懸念点を洗い出し
- 優先すべき要素を特定

**Phase 4: 決定**
- 投資額の決定
- 実行タイミング
- 記録すべき内容（INVESTMENT_PROFILE.md更新）
